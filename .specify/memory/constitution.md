# Claude Dashboard Constitution

## Core Principles

### I. Cross-Platform First
Every feature must work on Windows 11 and Linux. Platform-specific code is isolated behind abstractions — never scattered through the codebase. If a feature can't be made cross-platform, it's gated by platform detection and degrades gracefully.

### II. Low Profile
The dashboard is a utility, not a main application. Minimal resource usage, small visual footprint, no unsolicited noise. The user controls when to look and when to act. No popups, no sounds, no notifications unless explicitly configured.

### III. Proven Stack
Python + Tkinter. Patterns per [jewzaam/standards](https://github.com/jewzaam/standards): dataclass settings with atomic JSON persistence, threaded polling with tkinter `after()` callbacks, lazy window creation, dynamic row management via grid. No framework churn — use what works.

### IV. Configuration Over Opinion
Row dimensions, colors, emojis, poll interval, always-on-top behavior — all configurable. Sensible defaults that work out of the box, but the user can change anything visual or behavioral.

### V. Incremental Delivery
Focus on critical capabilities ahead of nice-to-haves. Each user story is independently testable and deployable. New features layer on without rearchitecting.

### VI. Simplicity
Start simple, YAGNI. No abstractions for single-use code. No feature flags. No plugin architecture unless the project grows to need one. Three similar lines of code beat a premature abstraction.

## Constraints

- No elevated privileges (no sudo/su)
- No external services or network calls — local-only operation
- Must not interfere with running Claude sessions (read-only observation for MVP)
- Must handle sessions in VS Code (Git Bash), Linux terminal emulators, and `screen`

## Development Workflow

- All documentation in Markdown, in `docs/` (project-wide) and `specs/` (per-feature)
- Coding standards follow [jewzaam/standards](https://github.com/jewzaam/standards) (Tkinter standards pending — will be derived from this project)
- Attribution in generated files: `Generated By: Claude Code (Claude Opus 4.6)`

## Governance

Constitution supersedes ad-hoc decisions. Amendments require updating this document.
